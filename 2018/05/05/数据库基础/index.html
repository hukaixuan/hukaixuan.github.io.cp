数据库基础

2018-05-05

SQL

  • SQL可分为两部分：DML(数据操作语言)、DDL(数据定义语言)
  • 查询、更新指令构成DML：SELECT/UPDATE/DELETE/INSERT INTO
  • DDL使我们能创建删除表格、定义索引、规定表间链接、施加表间约束：
    • CREATE DATABASE
    • ALTER DATABASE
    • CREATE TABLE
    • ALTER TABLE
    • DROP TABLE
    • CREATE INDEX
    • DROP INDEX
  • SELECT去重：SELECT DISTINCT 列名 FROM 表名
  • WHERE 子句：SELECT 列名 FROM 表名 WHERE 列 运算符 值
    • 运算符有：= 、 <> (不等于)、 >、 <、 >=、 <=、 BETWEEN、LIKE
    • SQL 使用单引号来环绕文本值（大部分数据库系统也接受双引号）。如果是数值，请不要使用引号。
  • AND / OR
  • ORDER BY: ASC(升序、默认) / DESC(降序)
    • SELECT Company, OrderNumber FROM Orders ORDER BY Company ASC, OrderNumber DESC;
  • INSERT INTO
    • INSERT INTO 表名 VALUES (值1,值2,…)
    • 指定插入数据的列：INSERT INTO table_name(列1, 列2, …) VALUES(值1,值2, …)
  • UPDATE: 修改表中的数据 UPDATE 表名 SET 列名=新值 WHERE 列名=某值
  • DELETE: 删除表中的行 DELETE FROM 表名 WHERE 列名=某值
    • 删除所有行：DELETE FROM 表名 或 DELETE * FROM 表名 ，在不删除表的情况下删除所有行，意味着表的结构、属性和索引都是完整的。
  • LIMIT [offset,] number
  • LIKE / NOT LIKE:
    • 通配符：
      • % 代替一个或多个字符
      • _ 仅代替一个字符
      • （MYSQL中不是，而是使用RLIKE）[charlist] 字符列表中的任何单一字符 、 [^charlist] 或 [!charlist] 不在字符列表中的任何单一字符
  • IN: 允许在where子句中规定多个值 SELECT 列名 FROM 表名 WHERE 列名 IN (值1,值2, …)
  • BETWEEN: BETWEEN … AND … 用在 WHERE 子句中，选取介于两个值之间的数据范围，可以是数值、文本或日期
    • SELECT column_name(s) FROM table_name WHERE column_name BETWEEN value1 AND value2
    • MYSQL 中包括value1和value2的值
    • 不在范围内 not between
  • Alias 别名：可以为列名称和表名称指定别名
    • 表的Alias： SELECT column_name(s) FROM table_name AS alias_name
    • 列的Alias： SELECT column_name AS alias_name FROM table_name
  • JOIN
    • INNER JOIN 等同于 JOIN： 只返回匹配行 SELECT column_name(s) FROM table1_name INNER JOIN table2_name ON table1_name.column=table2_name.column
    • LEFT JOIN 会从左表(table1_name)返回所有行，即使有些行在右表(table2_name)中没有匹配的行 SELECT column_name(s) FROM table1_name LEFT JOIN table2_name ON table1_name.column=table2_name.column
    • RIGHT JOIN 会从右表(table2_name)返回所有行，即使有些行在左表(table1_name)中没有匹配的行 SELECT column_name(s) FROM table1_name RIGHT JOIN table2_name ON table1_name.column=table2_name.column
    • FULL JOIN 只要其中某个表存在匹配，FULL JOIN 就会返回行
  • UNION 用于合并两个或多个SELECT结果集，UNION内部的SELECT语句必须拥有相同数量的列，列有相似的数据类型，列的顺序相同。
    • SELECT column(s) FROM table1 UNION SELECT column(s) FROM table2 注意: UNION会去掉重复的值，允许有重复的值用 UNION ALL, UNION 结果集中的列名总是等于 UNION 中第一个 SELECT 语句中的列名
  • SELECT INTO: 从一个表中选取数据插入另一个表中，可用于创建表的备份 SELECT * INTO new_table [IN externaldatabase] FROM old_table 或只插入特定的列： SELECT column(s) INTO new_table [IN externaldatabase] FROM old_table
    • eg: SELECT * INTO Persons IN 'Backup.mdb' FROM Persons
    • 也可使用WHERE子句或表连接
  • CREATE DATABASE database_name;

  • CREATE TABLE:

  • data_type包括：

    • integer(size)/int(size)/smallint(size)/tinyint(size) 仅容纳整型，括号内规定数字的最大位数
    • decimal(size, d)/ numeric(size, d) 容纳带小数的数字，size规定数字的最大位数，d规定小数点右侧部分的最大位数
    • char(size) 容纳固定长度的字符串，size规定字符串的长度
    • varchar(size) 容纳可变长度的字符串，size规定字符串的最大长度
    • date(yyyymmdd) 容纳日期

  • SQL约束（Consistraints）: 限制加入表的数据的类型，可以在创建表的时候规定约束（CREATE TABLE），也可以在表创建之后，（ALTER TABLE）

    • NOT NULL 强制字段始终包含值。这意味着，如果不向字段添加值，就无法插入新记录或者更新记录。
    • UNIQUE 唯一标识数据库表中的每条记录
    • PRIMARY KEY 拥有自动定义的UNIQUE约束，一个表可以有多个UNIQUE约束，只能有一个PRIMARY KEY约束
    • FOREIGN KEY 一个表中的FOREIGN KEY指向另一个表中的PRIMARY KEY，用于预防破坏表之间连接的动作，同时防止非法数据插入外键列，因为它必须是它指向的另一个表中的PRIMARY KEY的值之一 eg: FOREIGN KEY (Id_P) REFERENCES Persons(Id_P) 命名并使用多个FOREIGN KEY：CONSTRAINT fk_PerOrders FOREIGN KEY (Id_P) REFERENCES Persons(Id_P)
    • CHECK 用于限定列中值的范围 eg: CHECK (Id_P>0)
    • DEFAULT 向列中插入默认值
  • CREATE INDEX：在表中创建索引，可以在不读取整个表的情况下使数据库应用更快地查找数据（更新一个包含索引的表需要比更新一个没有索引的表更多的时间，这是由于索引本身也需要更新。因此，理想的做法是仅仅在常常被搜索的列（以及表）上面创建索引）

CREATE VIEW view_name AS
SELECT column_name(s)
FROM table_name
WHERE condition

SELECT column_name, aggregate_function(column_name)
FROM table_name
WHERE column_name operator value
GROUP BY gp_column_name

SELECT column_name, aggregate_function(column_name)
FROM table_name
WHERE column_name operator value
GROUP BY column_name
HAVING aggregate_function(column_name) operator value

SELECT Customer,SUM(OrderPrice) FROM Orders
GROUP BY Customer
HAVING SUM(OrderPrice)<2000
```

  • UCASE() 转大写
  • LCASE() 转小写
  • MID(column, start[,length]) 从文本字段中提取字符
  • LEN()
  • ROUND(column_name, decimals) decimals 规定要返回的小数位数
  • FORMAT(column_name, format)

REDIS

  • 五种存储类型：字符串STRING、列表LIST、集合SET、散列表HASH、有序集合ZSET
  • 持久化
  • STRING支持：SET/GET/DEL
  • LIST支持：LPUSH/RPUSH/LPOP/RPOP/LINDEX(取出index位置的元素)/LRANGE(列出某个范围内的元素)
  • SET支持：SADD(返回1表示成功添加，0表示已经存在)/SRAM(移除元素)/SISMEMBER/SMEMBERS
  • HASH支持：HSET/HGETALL(获取所有的键值对)/HDEL/
  • ZSET有序集合：键为成员，值为分值(浮点数，依分值排序)
    • ZADD：zadd zset-key socre member
    • ZRANGE
    • ZRANGEBYSCORE
    • ZREM
